Key Design Principles

Click an approach to select it

A

Full System Colors — Zero Custom Colors

Use only SwiftUI semantic colors: .primary, .secondary, .tertiaryLabel, .controlAccentColor. No custom hex values at all. Buttons use .bordered and .borderless native styles. The app looks exactly like a system popover — indistinguishable from macOS Settings or Spotlight.

Pros

  • Zero maintenance — automatically adapts to future macOS themes
  • Perfect dark/light mode for free
  • Feels invisible, like part of the OS

Cons

  • Very plain — no personality at all
  • Accent color is whatever the user set in System Settings
B

System Colors + Subtle Accent

System semantic colors everywhere except a single app accent color (e.g., a warm orange or teal) used sparingly on the active timer ring/progress and the primary action button. Everything else — text, backgrounds, dividers, secondary buttons — uses system colors. Think: macOS Calculator with one branded touch.

Pros

  • Native feel with a hint of identity
  • Still adapts to dark/light mode automatically
  • The accent draws the eye to the timer — the one thing that matters

Cons

  • Slightly more to maintain (one color)
C

Vibrancy Material — Translucent Panel

Use .ultraThinMaterial or .regularMaterial for the panel background — the desktop wallpaper bleeds through like Control Center or Notification Center. System semantic colors for text and controls. The panel itself becomes the design statement.

Pros

  • Most "Apple" feel — matches Control Center, widgets, Notification Center
  • Gorgeous with any wallpaper
  • Dark mode vibrancy is particularly beautiful

Cons

  • Timer text readability can suffer on busy wallpapers
  • Needs careful contrast testing
