**TimerEngine.swift** — Add a public `startBreak()` method to the Public API section (after `stop()`):
```swift
func startBreak() {
    ticker?.invalidate()
    ticker = nil
    currentMode = .break_
    targetDuration = breakDuration * 60
    beginSession()
}
```
This stops any running work session and immediately starts a break using the user's configured break duration. It reuses `beginSession()` which sets `timerState = .running`. Note: `startBreakWithDuration(seconds:)` already exists for testing — this new method is the user-facing equivalent using the stored `breakDuration` preference.

**FloatingPanelWindow.swift** — NSPanel subclass with these CRITICAL init-time requirements:
- Style mask MUST include `.nonactivatingPanel` in the `super.init()` call (cannot be set after init — this is a documented AppKit bug where the window server tag is not updated post-init)
- `isFloatingPanel = true` set in init body before panel is shown
- `level = .floating`
- `hidesOnDeactivate = false` (panel stays visible when app loses focus — essential because app runs in `.accessory` mode)
- `collectionBehavior = [.canJoinAllSpaces, .fullScreenAuxiliary]` (both flags required: `.canJoinAllSpaces` makes panel appear in every Space, `.fullScreenAuxiliary` overlays fullscreen app Spaces)
- `isMovableByWindowBackground = true`
- Hide all standard window buttons (close, miniaturize, zoom) — toggle is the only show/hide mechanism
- `titleVisibility = .hidden`, `titlebarAppearsTransparent = true`
- Accept `timerEngine: TimerEngine` parameter, host `ControlsView` via `NSHostingView(rootView:)`
- `setFrameAutosaveName("FloatingPanel")` for position persistence across restarts
- Override `canBecomeKey` -> `true` (allows button interaction), `canBecomeMain` -> `false` (does not become main window)
- Content rect: `NSRect(x: 0, y: 0, width: 240, height: 200)`
- Full style mask: `[.titled, .closable, .nonactivatingPanel, .fullSizeContentView]`

**ControlsView.swift** — SwiftUI view for the panel content:
- Takes `@ObservedObject var timerEngine: TimerEngine` (NOT @StateObject — must share the existing engine instance)
- Mode label at top: "Ready" / "Working" / "Paused" / "Break" (same logic as MenuBarView.modeLabel)
- Large monospaced countdown: `.font(.system(size: 56, weight: .light, design: .monospaced))` with `.monospacedDigit()`
- Control buttons with this conditional logic:
  - **Start** button: shown when `idle` or `paused` — calls `timerEngine.start()`
  - **Pause** button: shown when `running` — calls `timerEngine.pause()`
  - **Break** button: shown when `idle` or when `running` with `currentMode == .work` — calls `timerEngine.startBreak()`. This lets users skip remaining work time and jump straight to break.
  - **Stop** button: shown when `running`, `paused`, or `onBreak` — calls `timerEngine.stop()`
- Start button gets `.keyboardShortcut(.defaultAction)`
- `.controlSize(.large)` on the button HStack
- Padding: 20pt, frame: `minWidth: 220, idealWidth: 240, maxWidth: 300`
- Private `formatTime(_:)` and `modeLabel` helpers (same as MenuBarView)
